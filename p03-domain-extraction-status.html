✅ Catalog — Extracted & Running

The Catalog domain was the first extraction and served as the template for all subsequent domains. It has been running independently in K8s since February 12, 2026.

What Was Extracted Catalog domain details

Scope

  • Product Listing Pages (PLP) — category browse, search results grid
  • Product Detail Pages (PDP) — product info, images, specs, reviews
  • Catalog BFF — 8 API endpoints serving product data
  • 14 React components specific to catalog functionality

What Remained in Monolith

  • Cart integration points — “Add to Cart” button uses message bus to Cart domain
  • Search typeahead — owned by Search domain, Catalog consumes results

Results

  • 347 unit tests passing, 28 integration tests passing
  • Bundle size reduced 12% from monolith baseline
  • Deploy time: 4 minutes (was 18 minutes in monolith)
  • Zero production incidents since extraction
Industry Standard: First Domain as Template The first domain extraction creates the playbook for all others: Helm charts, CI/CD pipeline configs, monitoring dashboards, and runbooks. Every subsequent extraction follows this proven template.

✅ Shared/Core — Isolated & Published

The Shared/Core layer — header, footer, authentication, and cross-cutting utilities — was isolated and published to the CDN as independently loaded modules.

What Was Isolated Shared/Core details

Scope

  • ml-header — Global navigation header with user context
  • ml-footer — Site footer with legal links and site map
  • ml-auth-provider — Authentication context and token management
  • ml-breadcrumb — Navigation breadcrumbs
  • 12 utility functions (formatting, validation, analytics hooks)

How It Works

Shared/Core is loaded once via the import map and consumed by all domains. It does not have its own deployment pipeline — it deploys to CDN on merge to main, and domains pick up the latest version automatically via import map resolution.

🔵 Search — 60% Complete

Search extraction is progressing well. Typeahead and basic PLP search are decoupled. Faceted search and personalization hooks are the next focus areas.

What’s Done vs. What Remains Search extraction details

✅ Completed

  • Search repository created from monolith copy
  • Non-Search code stripped
  • Search BFF created with 3 of 5 endpoints operational
  • Typeahead search fully decoupled and tested
  • PLP search results rendering independently
  • 94 unit tests passing

🔵 Remaining

  • Faceted search filters (category, brand, price range) — estimated 5 days
  • Personalization hooks (recently viewed, purchase history weighting) — estimated 3 days
  • Search BFF endpoints 4 & 5 (autocomplete suggestions, search analytics) — estimated 2 days

Blockers

Personalization requires Account BFF data (POC 3 found this dependency). Workaround: Search BFF will call Account BFF directly via internal service mesh until Account is fully extracted.

Extraction Order Rationale

Domains are extracted in dependency order — foundational domains first, dependent domains after. This minimizes cross-domain coordination during extraction.

Domain dependency map — green = extracted, blue = in progress, gray = planned
Industry Standard: Dependency-Ordered Extraction Extracting foundational domains first (Shared/Core, Catalog) creates stable interfaces that dependent domains can rely on. This is the same topological-sort approach used by every major monolith decomposition playbook.
